# 【期末最終課題】「空気を読む」AIチャットボット開発プロジェクト

## 1. プロジェクトの目的
これまで習った JavaScript の基礎と、GitHub Copilot（生成AI）を活用して、会話の文脈を覚え、状況によって返答を変える「賢いチャットボット」を作ります。

重要なのは「AIに全部やらせる」ことではなく、みなさんが設計者（ディレクター）となって仕様を決め、Copilotを道具として使いこなすことです。

---

## 2. 完成イメージ（何を作ればいいか）
- 会話の中からユーザーの「名前」「好きなもの」などを抽出して記憶する。
- ボットが内部に「ステータス（例：好感度、機嫌、疲れ）」を持つ。
- 同じ入力でもステータスによって返答が変わる。

例：頑固なラーメン屋の店主ボット
- 記憶：客が前回頼んだラーメンの「硬さ」や「スープ濃さ」を覚える。
- ステータス：「忙しさ（0〜100）」を持つ。
- 変化：
  - 忙しさが低い → 「いらっしゃい！今日はおすすめは…」
  - 忙しさが高い → 「後で来てくれ…今忙しい」

---

## 3. 開発の絶対ルール（守ってください）
1. 配布された `index.html` の構造（ID・クラス名）は絶対に変更しないこと。  
   - HTML を勝手に書き換えたものは評価対象外です。
2. 配布された `script.js` にある「空の関数」を埋める形式で実装すること。  
   - 例：`processBotLogic` / `saveName` / `updateMood` などの中身を作る。
3. 設計→実装→相互デバッグの順で進めること（以下参照）。

---

## 4. 作業ステップ（推奨の進め方）

### STEP 1：設計（まず紙に書く）
コードを書く前に必ず設計を行ってください。提出必須です。
- キャラクター設定（例：ツンデレ執事、やさしい先輩、無愛想な店員）
- どんな情報を記憶するか（例：userName, favoriteFood）
- どんなステータスを持つか（例：mood, energy, friendship）
- 会話の基本フロー（初対面→名前を聞く→覚える→数ターン後に参照する）

> 設計例（簡単）
> - キャラ：反抗期の弟
> - 記憶：userName, favoriteGame
> - ステータス：annoyance (0-100)
> - ルール：相手が褒めるとannoyance-10、悪口を言うと+20

---

### STEP 2：実装（Copilotを使って進める）
`script.js` のコメントに従って、空の関数の中身を実装します。Copilotを使う際は具体的に指示してください。

- 悪い指示（NG）
  - 「いい感じの会話ボット作って」
- 良い指示（OK）
  - 「`updateMood` 関数の中で、入力テキストに 'バカ' が含まれていたら `botState.mood -= 10` するコードを書いて」
  - 「`responseData` のデータ構造に合わせて、ツンデレキャラ用の会話パターンを5つ追加するオブジェクトを生成して」

実装中は必ず動作テストを行い、意図した仕様（記憶・ステータス反映）が行われているか確認してください。

---

### STEP 3：相互デバッグ（ペアワーク）
完成したら別の生徒と作品を交換してテストします。相互デバッグの目的は次のとおりです。
- 自分で気づかなかった会話の破綻を見つける。
- 設計通りに記憶やステータスが動いているか検証する。
- バグをフィードバックして改良する。

相互デバッグでのチェック項目例：
- 名前を教えるとその後参照されるか
- 同じ挨拶でステータスによって返答が変わるか
- 意図しない入力でエラーにならないか

---

## 5. 提出物（必須）
1. ソースファイル一式（`index.html`, `style.css`, `script.js`）  
   - ※index.html は変更禁止だが、提出は必須
2. 設計書（PDFまたはMarkdown）  
   - キャラクター、持つパラメータ、会話シナリオ（簡単なフローチャート）
3. 開発レポート（MarkdownまたはPDF）  
   - Copilotとの対話ログ（主なプロンプトと生成結果の抜粋）  
   - Copilotに頼んで成功した例・失敗した例とその理由  
   - 自分が理解している実装の説明（重要な関数の解説）
4. 相互デバッグ記録（誰と交換したか、見つかった不具合と対応内容）

---

## 6. 評価のポイント
- 仕様（HTMLの構造、関数名など）を守って実装しているか
- 正規表現で情報抽出ができているか（名前や好きなもの等）
- ステータスに応じて会話が変化しているか
- Copilotをただの「コピー要員」として使わず、指示・検証・修正ができているか
- 設計書とレポートに論理的な説明があるか

---

## 7. Copilotへの指示のコツ（例）
- NG例（曖昧）
  - 「良い感じのボット作って」
- OK例（具体的）
  - 「`responseData` 配列に `{ keyword: /私の名前は(.*)です/, action: 'saveName', response: '{name}さんだね' }` のようなオブジェクトを追加するコードを5つ作って。」

---

## 8. 注意事項
- HTML の ID やクラス名を変えたものは受け付けません。評価対象から外れる可能性があります。
- 他人のコードを丸写しすることは不可です。Copilotの生成物をそのまま提出するのではなく、必ず理解してから提出して下さい。
- 分からない点があれば授業時間に質問してください。

---

## 9. 最後に
設計力と指示力（プロンプト設計）は、これからの開発でとても重要なスキルです。Copilotは強力な道具ですが、道具は使う人の考え方で結果が大きく変わります。良い設計を作って、賢いボットを育ててください！
